# E-commerce Order Pipeline Demo Project
## Overview
This demo project showcases the capabilities of Genie Code, a recently introduced feature by Databricks, in building AI-powered data applications. The project demonstrates an e-commerce order pipeline with real-time inventory updates, Slowly Changing Dimension (SCD) Type 2 tracking, and daily sales aggregation. The pipeline utilizes Genie Code, Delta Lake, and SQL Warehouse, along with window functions for time-based analysis and low-stock alerting using MERGE for upserts.

## Why this feature matters
Genie Code matters because it bridges the gap between data engineering and AI, providing a unified platform for data teams to work together and drive business outcomes. By leveraging Genie Code, users can build, deploy, and manage data-driven solutions with ease, enabling real-time decision-making and improved business performance.

## Prerequisites
To run this demo project, you will need:
* A Databricks account with access to Genie Code, Delta Lake, and SQL Warehouse
* Basic knowledge of Databricks, SQL, and data engineering concepts
* Intermediate level of complexity

## How to run
To run this demo project, follow these steps:
1. **Notebook**: Open the provided notebook and run each cell in sequence. The notebook will create the necessary tables, load sample data, and execute the Genie Code pipeline.
2. **SQL**: Run the provided SQL queries in a SQL Warehouse query editor to perform time-based analysis and low-stock alerting.

## Expected output
The expected output of this demo project includes:
* Real-time inventory updates with SCD Type 2 tracking
* Daily sales aggregation with window functions for time-based analysis
* Low-stock alerting using MERGE for upserts
* A unified view of the e-commerce order pipeline with AI-powered insights
