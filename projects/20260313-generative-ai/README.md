# Generative AI Demo Project: Text Classification Model
==============================================

## Overview
This demo project showcases the capabilities of Databricks' Generative AI feature in building and deploying a text classification model. The model is trained to classify customer feedback into predefined categories, demonstrating the potential of machine learning in natural language processing.

## Why this Feature Matters
The ability to generate human-like text and classify unstructured data has numerous applications in industries such as customer service, marketing, and healthcare. With Databricks' Generative AI capabilities, users can:

* Automate text analysis and classification tasks
* Improve the accuracy of sentiment analysis and customer feedback classification
* Enhance the overall customer experience through personalized responses and recommendations

## Prerequisites
To run this demo project, you need:

* A Databricks account with access to a cluster
* The Databricks ML runtime installed on your cluster
* A sample dataset of customer feedback (provided in the demo project)

## How to Run
This demo project consists of two parts: a notebook and a SQL script.

### Notebook
The notebook demonstrates the following steps:

1. Data ingestion and preprocessing
2. Model training and evaluation
3. Model deployment and testing

To run the notebook:

1. Import the notebook into your Databricks workspace
2. Attach the notebook to a cluster with the Databricks ML runtime installed
3. Run each cell in sequence to execute the code

### SQL
The SQL script demonstrates how to use the trained model to classify new customer feedback. To run the SQL script:

1. Create a new SQL query in your Databricks workspace
2. Copy and paste the SQL script into the query editor
3. Execute the query to classify new customer feedback

## Expected Output
The expected output of this demo project is a classified dataset of customer feedback, with each feedback text labeled as positive, negative, or neutral. The output will also include metrics such as accuracy, precision, and recall, which can be used to evaluate the performance of the model.
